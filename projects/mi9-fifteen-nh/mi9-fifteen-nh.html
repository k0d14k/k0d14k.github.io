Kali NetHunter Port for Xiaomi Mi 9 (cepheus)

ls -l index

Dec 14 01:10

cat introduction

Kali NetHunter is essentially a mobile penetration-testing distribution that turns a compatible Android device into a portable attack platform. It brings the capabilities of Kali Linux to smartphones.

Kali NetHunter is a penetration-testing framework composed of:

  • A custom kernel with patches tailored for mobile security assessments (e.g., HID gadget support, monitor mode on compatible chipsets, USB Arsenal).
  • A Kali Linux chroot or container providing the full suite of well-known security tools such as Nmap, SQLmap, Metasploit, Impacket, and the broader Kali toolset.
  • An Android application acting as a control center, used to launch tools, manage permissions, and expose modules like HID Attacks, MITM utilities, Wi-Fi toolkits, and more.
  • Physical-layer attack capabilities, including BadUSB/HID keyboard emulation, USB network spoofing, Evil AP setup, and wireless packet injection where supported by the hardware.

From a defensive security perspective, understanding NetHunter is valuable because it highlights the "mobile" side of the attack surface: USB gadgeting, NFC fuzzing, Android permission handling, kernel hardening, and the ways in which a compromised device can be leveraged as a pivot point.

In CTF or red-team scenarios, it becomes a compact attack platform, enabling network and device testing even on the move and allowing realistic "proximity-based" attack simulations that would be far less natural to execute from a desktop environment.

This guide explains how to start from a fully locked device (without root access) and transform it into a system with full NetHunter support, including a custom-patched kernel that provides the required compatibility with the framework.

The first place you might expect to find useful information is the official Kali website.

On the site, under the Mobile --> NetHunter section (excluding NetHunter PRO), you can find the list of supported devices. These devices, along with their recommended custom ROMs, are referred to as "listed devices" and feature native support with kernels patched directly by the Kali developers. All other devices not explicitly mentioned fall into the "generic devices" category.

As mentioned earlier, Kali NetHunter includes a chroot containing a root filesystem. Depending on the tools included in the installation package, there are different formats available: nano, minimal, and full.

But the real question is: "Why should I build a custom kernel when I could simply use the Kali installer?"

The answer may sound obvious, yet it took me much longer than expected to fully understand it and that's exactly why I decided to write this article. The main issue with the generic NetHunter installation is, in fact, its generic nature. Since it must run on any rooted device, it cannot account for all the hardware differences across models and therefore cannot guarantee compatibility with the hardware interfaces that make NetHunter such an effective tool for red teaming and physical-layer hacking.

Let's begin this journey with the modification of my Xiaomi Mi 9 (cepheus). Since it is not officially supported, it caused more than a few headaches, but also a great deal of satisfaction once I finally completed the process and understood the full potential of this framework.

Step 1 - Bootloader Unlock.

Warning

This procedure will erase all data on your device, so make sure to create a full backup beforehand.

Unlocking the bootloader is relatively straightforward, as Xiaomi officially supports this procedure. The required steps are:

  1. Open About Phone and tap repeatedly on MIUI Version to enable Developer Options.

  2. Once Developer Options are available, enable Debug USB, OEM Unlocking and open Mi Unlock Status. From there, add your Xiaomi account and link it to the device. The Debug USB option will allow us to issue commands via the Android platform-tools. If you are starting from zero you should need to install them.

  3. Download the Mi Unlock Tool, sign in with the same Xiaomi account, connect the device in fastboot mode, and initiate the unlock process.


Once the device has rebooted, return to the Developer Options menu. At this point, you should see a configuration similar to the following:

Step 2 - Flashing a custom recovery.

For this guide, I will use the PixelOS custom ROM, maintained by a developer known as @balgxmr. The first step, therefore, is to install the official custom recovery provided for this ROM: PixelOS Recovery.

Connect you device, allow debug usb and send following commands:

Warning

I'm a Linux user, if you are a Windows one your sintax could be a little bit different but you should solve by adding .exe at the end of the command eg. adb --> adb.exe

adb reboot-bootlaoder
fastboot flash recovery [recovery name]
fastboot boot [recovery name]

Info

[recovery name] referes to the name of the recovery file you downlaoded.

Step 3 - Flashing custom ROM and Magisk.

Once the custom recovery has been flashed, we can proceed with installing the custom ROM and the Magisk package required for root privilege management.

To do this, boot the device into recovery mode and, after wiping all data using the Wipe Data and Factory Reset options, navigate to Apply Update --> Aply from ADB and install both packages using adb sideload.

adb sideload [rom zip file]
adb sideload [magisk zip file]

Info

[rom zip file] referes to the name of the custom ROM package you downlaoded.
[magisk zip file] referes to the file you get by copying the magisk apk and renaming it from .apk into .zip. It is recommended to use Magisk 28.1

At this stage, the device is fully rooted and we can proceed with porting our NetHunter build. But first, we need a recovery that supports the dynamic partitions of the Cepheus. The TWRP EROFS comes to the rescue (which I obtained for you from the PixelOS Telegram channel) and is available for download here.
adb reboot-bootloader
fastboot flash recovery recovery.img
fastboot boot recovery.img

cat nethunter_kernel_builder

Setting aside how a kernel actually operates on Android, the main reasons why users create custom versions of their kernel are essentially two:

  1. To modify or optimize existing functionalities;
  2. To extend the supported capabilities with new features;

In the case of the NetHunter port, as mentioned earlier, it is necessary to rely on internal hardware, external peripherals, and specialized devices to perform certain types of tests involving technologies such as Bluetooth, CAN, and Wi-Fi. The official Kali documentation provides extensive guidance on how to configure the kernel to ensure NetHunter compatibility, but in practice several issues may arise (issues that I managed to address during my research).

I do not want this article to become overly long, technical, or tedious for readers who are not specialists. However, I would like to focus on the key points that enabled the creation of this project.

Everything starts from the base configuration. The choice of the PixelOS custom ROM was intentional: I selected it both because the maintainer is quite active and because the kernel source code is publicly available on GitHub.

The same process could have been carried out using the official Xiaomi kernel, but unfortunately I was unable to locate the complete source code. Many manufacturers, despite Android kernels being subject to the GPL license, do not release all the components they have developed for their devices, making the process even more challenging.

Once the kernel source code is obtained, the next step is compiling it. To accomplish this, I created a Docker container containing all the required build tools.

The practical section of this article focused on understanding how to make the different kernel components operate as intended. The basic NetHunter support can be implemented simply through configuration, while the most challenging aspect was enabling driver support for the TP-Link TL-WN722N v2 Wi-Fi adapter. This device is not designed for security testing (and it does not natively support monitor mode or packet injection), but it can be adapted using custom drivers developed by Aircrack for the Realtek RTL8188EUS chipset.

I consolidated all the adaptation work and the porting of the Cepheus kernel into a setup script that prepares the source code for use with the Kali NetHunter Kernel Builder.

As a result, in this guide I will not focus on how to apply the individual code modifications (since I have already made them for you), but instead on how to execute my script (and we will actually see that you won't even need to run it).

git clone https://github.com/k0d14k/mi_9_cepheus_kali_nethunter_setup
cd mi_9_cepheus_kali_nethunter_setup
docker compose up -d
docker exec -it k0d14k-nh-builder bash
./runner.sh
These commands are all you need to run my script. Just make sure that Git and Docker are properly installed and configured. Once executed, the script will clone my kernel fork and automatically apply all the required patches. Your only task will be to use the Kali NetHunter Kernel Builder script for a few operations that we will examine in detail later on.

Do not worry if cloning some components takes longer than expected; this is simply because the projects are quite large and therefore relatively heavy.

This is how your terminal will appear once the script is ready to launch the Kali one. I want to reiterate that I am running the entire process from a Linux distribution, and I strongly recommend that you do the same. None of the procedures described in this document (with the exception of the bootloader unlocking tools, which are Windows-specific) have been tested outside of Linux, and they may therefore not work correctly or could potentially cause issues.

Once you press ENTER, follow the steps below:

  1. In "4. Apply NetHunter kernel patches", select kernel version 4.14 and apply patches 4 and 5.
  2. In "2. Configure & compile kernel from scratch", choose the cepheus_defconfig file and, when prompted with "Edit the kernel config? [Y/n]", select Y to configure the kernel.
  3. The following screens will display the options to select and how to configure them.

Back --> General Setup

Device Drivers --> Network device support --> Wireless LAN

Go back until you reach the screen shown below; at that point, select No and let the script handle the remainder of the process automatically.

You can also answer No to the prompt "Save .config as nethunter_defconfig? [y/n]".

At this stage, all you need to do is generate the AnyKernel installation package using the "6. Create AnyKernel zip" command. Once the zip file has been created, reboot your device into recovery mode and flash it, enabling full NetHunter functionality on your system.

At the end of the process, you can use the E command to exit the Kali NetHunter Kernel Builder. My script will then resume its execution, saving the results in the output directory and removing all the source code.

cat nethunter_installer_builder

Once the build process is completed using a fork of the official repository, it is possible to upload the modified kernel and, by editing and building the corresponding devices.yml file, add a custom port to the list of devices supported by Kali NetHunter. The original commit can be reviewed at the following link.

After the merge request has been opened and approved, the Kali NetHunter Installer Builder repository can be used to create a custom installation package for cepheus directly with the official toolchain.

The following commands allow you to build the **Kali NetHunter Full** installation package using my port:

git clone https://gitlab.com/kalilinux/nethunter/build-scripts/kali-nethunter-installer
cd kali-nethunter-installer
./bootstrap.sh
python3 build.py --kernel cepheus-pos --fifteen --rootfs full

Once the installation package has been built, it is sufficient to upload the resulting ZIP to the device and install it as a Magisk module.

The porting process is complete. Since Kali NetHunter is effectively a Linux instance, it is assumed that the reader is already familiar with using Linux and knows how to install libraries and apply updates.
